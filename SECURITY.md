# Security Policy

## Supported Versions

The `main` branch is the only actively supported version until the project starts publishing releases.

## Reporting a Vulnerability

Please do not open public issues for security reports. Send a private report to the project maintainers through GitHub security advisories once the repository is published.

Include:

- Affected component.
- Reproduction steps.
- Impact.
- Suggested fix, if known.

## Configuration Notes

- Do not commit secrets.
- Keep production CORS origins restricted to trusted domains.
- Validate deployment-specific environment values before releasing.
